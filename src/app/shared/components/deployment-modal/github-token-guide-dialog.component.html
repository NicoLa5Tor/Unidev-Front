Token de acceso

Cómo obtener un token de GitHub

¿Qué es?

Un Personal Access Token (PAT) es una credencial que reemplaza a su contraseña cuando una herramienta externa necesita acceder a su cuenta de GitHub. Lo necesitamos únicamente para clonar repositorios privados. Si su repositorio es público, deje el campo vacío.

Dos tipos de token: ¿cuál usar?

GitHub ofrece dos tipos de PAT. Ambos funcionan con UniDev, pero recomendamos el primero:

Fine-grained token (recomendado)

  • Prefijo: github_pat_…
  • Se puede limitar a un solo repositorio.
  • Permisos granulares (por ejemplo, solo Contents: Read-only).
  • Fecha de expiración obligatoria.
  • Si se filtra, el daño está acotado al repositorio seleccionado.

Classic token (funciona, pero no se recomienda)

  • Prefijo: ghp_…
  • El scope repo otorga acceso a todos sus repositorios privados, incluso a los de organizaciones donde sea miembro.
  • Si se filtra, el atacante puede leer y modificar toda su actividad en GitHub.
  • Úselo únicamente si su cuenta no permite generar fine-grained tokens (por ejemplo, en algunas organizaciones empresariales que aún no los han habilitado).

En la siguiente sección guiamos la creación del tipo recomendado (fine-grained).

Pasos para generar un fine-grained token

  1. Ingrese a github.com/settings/tokens (o desde GitHub: foto de perfil → Settings → Developer settings → Personal access tokens → Fine-grained tokens).
  2. Haga clic en "Generate new token".
  3. Asigne un nombre descriptivo, por ejemplo: UniDev deployment.
  4. En Expiration, seleccione un período corto (por ejemplo 7 o 30 días). Este token solo se usa para clonar una vez.
  5. En Repository access, elija "Only select repositories" y seleccione únicamente el repositorio que va a desplegar.
  6. En Repository permissions, busque Contents y cámbielo a Read-only. Es el único permiso necesario.
  7. Haga clic en "Generate token" y copie el valor que aparece (empieza con github_pat_…).
  8. Pegue el token en el campo "Git token" del formulario.

Importante

  • El token se muestra una sola vez. Si lo pierde, debe generar uno nuevo.
  • Nunca lo comparta ni lo suba a un repositorio.
  • El token se cifra con AES-256-GCM antes de almacenarse en nuestra base de datos.
  • Cuando termine la prueba, puede revocarlo desde la misma página de GitHub.
¿Usa GitLab o Bitbucket?

GitLab: Preferences → Access Tokens, scope read_repository.

Bitbucket: Personal settings → App passwords, permiso Repositories: Read.

El procedimiento es similar: cree un token con permiso de solo lectura del repositorio y péguelo en el formulario.
